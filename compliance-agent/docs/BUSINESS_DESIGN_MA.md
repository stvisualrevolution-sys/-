# 売却前提の具体ビジネス設計（運送業コンプライアンスAI）

## 1. 目標KPI（12ヶ月）
- ARR: 5,000万円
- 営業利益: 5,000万円（開発費回収後）
- チャーン率: 月次 < 2.0%
- Gross Margin: > 80%
- 監査対応レポート提出企業比率: > 70%

## 2. 価格戦略（売却しやすいSaaS指標を作る）
- Starter: 9.8万円/月（20ドライバーまで）
- Growth: 19.8万円/月（80ドライバーまで）
- Enterprise: 39.8万円/月〜（API連携・SLA）

年契約は2ヶ月分割引、解約防止のためオンボーディング費を別建て（15〜30万円）。

## 3. 買い手が評価する“防御力”
1. 監査証跡（改ざん耐性）
2. ルール更新運用（法改正に追従）
3. 低チャーン運用（通知+是正ワークフロー）
4. 導入スピード（CSV即日開始→API連携拡張）

## 4. 直近で実装済み/優先実装
### 実装済み
- マルチテナント認証
- 判定/通知
- CSV取込
- 月次レポート
- 監査イベントハッシュチェーン
- KPIサマリーAPI

### 次優先（売却価値を上げる）
- Stripe請求連携（MRRの可視化）
- 監査レポートPDF署名
- 外部通知の再送制御（DLQ）
- データルーム自動生成（監査・KPI資料）

## 5. 30社獲得までの営業設計
- ターゲット: 地場中堅運送（20〜200台）
- 初回訴求: 「違反予防」「監査時の証跡」「管理者工数削減」
- オファー: 30日トライアル + 初期設定代行

## 6. M&A出口戦略
- 候補: デジタコベンダー、運送向け基幹SaaS、労務SaaS、物流SI
- 売却前に整備する資料:
  - 月次ARR推移
  - チャーン分析
  - 顧客別粗利
  - セキュリティ運用手順
  - 監査証跡サンプル
